### Q3: 端口被占用怎么办？

**A**: 修改 `docker-compose.yml` 中的端口映射：

```yaml
ports:
  - "8003:8001"  # 修改后端端口
  - "8004:80"    # 修改前端端口
```

然后重启服务：
```bash
docker-compose down
docker-compose up -d
```

---

### Q4: 数据库连接失败？

**A**: 按以下步骤排查：

1. 检查 MySQL 是否启动
```bash
docker-compose ps
```

2. 查看 MySQL 日志
```bash
docker-compose logs mysql
```

3. 检查数据库配置
```bash
docker-compose exec backend env | grep DATABASE
```

4. 手动测试连接
```bash
docker-compose exec mysql mysql -uroot -proot123 -e "SHOW DATABASES;"
```

---

### Q5: 前端页面空白？

**A**: 

1. 清除浏览器缓存（Ctrl+Shift+R）
2. 检查浏览器控制台错误（F12）
3. 查看前端日志
```bash

```

## 🎯 功能使用

### Q6: 如何新建策略？

**A**: 
1. 进入策略中心
2. 点击"策略总览"
3. 点击"新建策略"按钮
4. 完成 5 步向导：
   - 步骤 1：基本信息
   - 步骤 2：业务单元划分
   - 步骤 3：成本模拟
   - 步骤 4：收益模拟
   - 步骤 5：确认保存

---

### Q7: 策略状态有哪些？

**A**: 
| 状态 | 说明 | 可编辑 |
|------|------|--------|
| 草稿 | 新建或编辑中 | ✅ |
| 模拟通过 | 模拟测试通过 | ✅ |
| 模拟失败 | 模拟测试失败 | ✅ |
| A/B 通过 | A/B 测试通过 | ✅ |
| A/B 失败 | A/B 测试失败 | ✅ |
| 已发布 | 已全量发布 | ❌ |
| 已归档 | 已归档 | ❌ |
| 已禁用 | 已禁用 | ✅ |

---

### Q8: 如何执行模拟测试？

**A**: 
1. 进入"策略实验室"
2. 选择"模拟测试"
3. 选择测试策略
4. 填写测试设置
5. 点击"启动模拟测试"
6. 等待测试完成
7. 点击"保存测试结果"

---

### Q9: 模拟测试和 A/B 测试有什么区别？

**A**: 

| 对比项 | 模拟测试 | A/B 测试 |
|--------|---------|---------|
| **目的** | 验证策略可行性 | 对比策略效果 |
| **数据** | 模拟数据 | 真实流量 |
| **周期** | 即时出结果 | 需要运行周期 |
| **风险** | 无风险 | 有业务风险 |
| **适用** | 策略初筛 | 策略验证 |

---

### Q10: 如何编辑已保存的策略？

**A**: 
1. 进入策略总览
2. 找到目标策略
3. 点击"编辑"按钮
   - 注意：已发布和已归档的策略不可编辑
4. 修改配置
5. 保存后状态变回"草稿"

## 🔐 权限相关

### Q15: 有哪些权限级别？

**A**: 
| 权限级别 | 说明 | 可执行操作 |
|---------|------|-----------|
| owner | 所有者 | 所有操作（包括删除） |
| editor | 编辑者 | 查看、编辑、测试 |
| viewer | 查看者 | 仅查看 |
| none | 无权限 | 无法访问 |

---

### Q16: 如何获取编辑权限？

**A**: 
1. 联系策略所有者
2. 或联系系统管理员
3. 在策略权限管理中添加用户

---

### Q17: 为什么我无法编辑某个策略？

**A**: 可能的原因：
1. 没有编辑权限（联系所有者）
2. 策略已发布（需要复制后编辑）
3. 策略已归档（需要复制后编辑）
4. 策略被其他人锁定（等待释放）

## 🔧 故障排查

### Q18: 点击按钮没有反应？

**A**: 
1. 检查浏览器控制台错误（F12）
2. 清除浏览器缓存（Ctrl+Shift+R）
3. 检查网络连接
4. 查看后端日志
```bash
docker-compose logs backend
```

---

### Q19: 页面加载很慢？

**A**: 
1. 检查网络延迟
2. 清除浏览器缓存
3. 检查服务器资源（CPU/内存）
4. 查看慢查询日志

---

### Q20: 保存失败，提示"无权限"？

**A**: 
1. 检查策略权限
2. 联系策略所有者授权
3. 或复制策略后编辑

---

### Q21: 测试保存失败？

**A**: 
1. 检查网络连接
2. 查看后端日志
3. 确认策略状态（草稿/模拟通过才能保存）
4. 检查数据库连接

---

### Q22: 系统报错"500 Internal Server Error"？

**A**: 
1. 查看后端日志
```bash
docker-compose logs backend | tail -100
```
2. 检查数据库连接
3. 检查磁盘空间
4. 重启服务
```bash
docker-compose restart
```
