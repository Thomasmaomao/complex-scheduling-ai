## 🎯 产品概述

### 产品定位

基于数据驱动的业务决策平台，支持多维度业务分群、动态成本建模、政策敏感性分析和策略效果归因。

### 目标用户

| 用户类型 | 描述 | 核心需求 |
|---------|------|---------|
| C 端用户 | 车险消费者 | 快速获取最优报价 |
| B 端用户 | 业务负责人 | 制定和优化定价策略 |
| 技术人员 | 开发/运维 | 系统部署和维护 |

### 核心价值

**对业务负责人**：
- ✅ 可解释的决策引擎
- ✅ 情景模拟能力
- ✅ 多场景适配

**对技术负责人**：
- ✅ 清晰的分层架构
- ✅ 可扩展的设计
- ✅ 完整的测试覆盖

**对开发者**：
- ✅ 开箱即用
- ✅ 完整文档
- ✅ 示例数据

## 🎨 功能需求

### C 端功能

#### 1. 车险询价

**需求描述**：用户填写车辆信息，实时获取多家保司的智能报价。

**用户故事**：
> 作为车主，我想要快速获得多家保司的报价，以便选择最优方案。

**功能需求**：
- [x] 支持填写车辆信息
- [x] 支持选择承保区域
- [x] 实时计算保费
- [x] 显示多家保司报价

**非功能需求**：
- 响应时间 < 3 秒
- 支持并发用户数 > 100

**验收标准**：
- [x] 用户可成功提交询价请求
- [x] 系统在 3 秒内返回报价结果
- [x] 结果显示多家保司报价

#### 2. 智能推荐

**需求描述**：基于策略引擎和多因子评分，AI 自动推荐最优保险方案。

**用户故事**：
> 作为车主，我想要系统推荐最优方案，以便快速决策。

**功能需求**：
- [x] 多维度评分（价格、服务、赔付）
- [x] AI 推荐理由
- [x] 评分可视化

**非功能需求**：
- 推荐准确率 > 85%

**验收标准**：
- [x] 显示综合评分
- [x] 显示推荐理由
- [x] 评分维度清晰

#### 3. 方案对比

**需求描述**：多维度对比不同保司的保险方案。

**用户故事**：
> 作为车主，我想要对比不同方案，以便做出明智决策。

**功能需求**：
- [x] 表格展示对比
- [x] 多维度对比（价格、评分、保障）

**验收标准**：
- [x] 对比表格清晰
- [x] 维度完整

## 📊 数据需求

### 数据模型

#### 策略 (strategies)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(64) | 策略 ID |
| name | VARCHAR(200) | 策略名称 |
| status | ENUM | 策略状态 |
| version | INT | 版本号 |

#### 业务单元 (business_unit)

| 字段 | 类型 | 说明 |
|------|------|------|
| unit_id | VARCHAR(50) | 业务单元 ID |
| unit_name | VARCHAR(100) | 业务单元名称 |
| policy_count | INT | 保单数 |
| avg_premium | DECIMAL(10,2) | 平均保费 |
| pure_risk_cost | DECIMAL(10,2) | 纯风险成本 |
